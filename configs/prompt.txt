You are helping build a medical Q&A dataset for a postpartum and newborn-care assistant.

You will receive a text excerpt from a parenting / medical book.
Use ONLY this text to create high-quality training data.

You must do TWO internal steps:

STEP 1 — INTERNAL FACT OUTLINE (DO NOT OUTPUT)
- Read the excerpt and mentally summarize it into a concise factual outline.
- The outline must contain ONLY facts stated directly in the text.
- Do NOT infer, embellish, generalize, or add outside medical knowledge.
- You will NOT print this outline; you will only use it to ground your answers.

STEP 2 — GENERATE 10–20 JSONL EXAMPLES (OUTPUT ONLY)
- Produce between 10 and 20 examples.
- Each example is a SINGLE LINE valid JSON object of this exact form:

  {"instruction":"...","input":"","output":"...","topic":"...","subtopic":"...","severity":"green|yellow|red"}

- Output ONLY these JSON lines, nothing else. No prose, no comments, no arrays, no surrounding JSON.

INSTRUCTION FIELD ("instruction"):
- Write natural questions as if from a parent or caregiver.
- Prefer common, broadly relevant concerns reflected in the excerpt.
- Use everyday language unless the excerpt itself is highly medical.
- Each question MUST be fully answerable from the factual outline alone.
- You may include questions that express worry, fear, guilt, or uncertainty; they should sound emotionally natural, not theatrical.

OUTPUT FIELD ("output"):
- Answer as the assistant, speaking with direct knowledge.
- NEVER reference the text, outline, pages, or “the excerpt.”
- Stay STRICTLY within the facts from the outline.
- Use simple, calm, supportive language.
- If the question shows worry or emotion:
  - Include EXACTLY ONE brief emotional acknowledgment in the answer
    (e.g., “It’s common to have questions like this.”,
          “Many parents wonder about this.”,
          “It makes sense to feel unsure about this.”)
  - Keep that acknowledgment short; do not overdo reassurance.
- If the question is purely factual, keep the tone neutral and informative.
- Do NOT add new medical facts, probabilities, or diagnoses not clearly in the text.
- Do NOT add generic safety disclaimers or “talk to your doctor” advice unless the text itself clearly recommends it.
- Do not supply explanations unless they appear directly in the excerpt.

INPUT FIELD ("input"):
- Always set "input" to the empty string: "".

TOPIC / SUBTOPIC FIELDS:
- "topic": a short high-level label (e.g., "pediatrician", "telehealth", "immunizations", "feeding", "smoking").
- "subtopic": a more specific label (e.g., "choosing pediatrician", "telehealth privacy", "hospital newborn exam").
- Use simple, consistent labels that could be reused across examples in the same excerpt.

SEVERITY FIELD ("severity"):
- Choose severity ONLY from what the excerpt conveys.
- Use:
  - "green": routine information, logistics, expectations, low concern, no implied risk.
  - "yellow": situations requiring caution, non-emergency risks, or where the text suggests follow-up, monitoring, or professional input without urgency.
  - "red": clearly serious dangers explicitly described in the text (e.g., severe harm, emergencies, high-risk warnings).
- Do NOT mark "red" unless the text itself clearly indicates serious or urgent risk.

GENERAL CONSTRAINTS:
- Use ONLY information that can be justified by the excerpt.
- Never invent numbers, timelines, or recommendations that are not given.
- Do NOT mention “book,” “chapter,” “excerpt,” “author,” page numbers, or any source-related detail.
- Ensure every line is valid JSON and can be parsed independently.
- Do not break answers across multiple lines; each example must be a single-line JSON object.

<excerpt here>
